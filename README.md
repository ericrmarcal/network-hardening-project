# Avaliação de Risco e Hardening de Rede - Mídia Social

## Descrição do Cenário
[cite_start]Este projeto simula a atuação de um Analista de Segurança em uma organização de mídia social que sofreu uma violação de dados[cite: 1, 2]. [cite_start]O objetivo foi identificar vulnerabilidades críticas e propor métodos de **Network Hardening** para proteger as informações pessoais dos clientes e evitar novos ataques[cite: 1].

## Vulnerabilidades Identificadas
[cite_start]Durante a inspeção, foram detectadas quatro falhas principais[cite: 1]:
* Funcionários compartilhando senhas.
* Senha de administrador do banco de dados configurada como padrão.
* Firewalls sem regras de filtragem de tráfego.
* Ausência de Autenticação Multifatorial (MFA).

## Soluções de Hardening Propostas
Com base nas melhores práticas de segurança, foram recomendadas as seguintes ações:
1. **Manutenção de Firewall:** Implementação de regras de filtragem para controle de tráfego de entrada e saída.
2. **Gerenciamento de Senhas e MFA:** Configuração de políticas de senhas fortes e obrigatoriedade de um segundo fator de autenticação para todos os acessos.

## Ferramentas Utilizadas
* Framework de Hardening de Rede (Google Cybersecurity Certificate).
* Relatório de Avaliação de Risco de Segurança.
